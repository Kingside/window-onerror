window.onerror

Documentation on the onerror property of window, and how it defers across browsers and browser versions.

Current WHATWG spec [link]

window.onerror = function(message, filename, lineno, colno, error) {
  // custom handling here
}
  • message: the error message (usually the Error object's message property)
  • filename: the absolute URL of the script in which the error originally occurred
  • lineno: line number (1-based) where the error occurred in the script
  • colno: column number (1-based) where the error occurred in the script
  • error: object representing the error (e.g. the exception object in the case of an uncaught DOM exception).

Return true from the function to prevent default browser error handling (such as showing a message in the console, or popping a debug dialog in old versions of IE).
